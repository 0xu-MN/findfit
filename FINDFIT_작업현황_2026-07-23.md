# FindFit 작업 현황 정리 (2026-07-23)

이 문서는 이번 세션(2026-07-22 ~ 2026-07-23)에 진행한 작업을 정리한 인수인계 문서입니다.

## 1. 인증/온보딩

### 해결된 근본 원인
- **모든 회원가입(이메일/구글/카카오)이 실패하던 버그**: `handle_new_user()` 트리거 함수가 `SECURITY DEFINER`인데 `search_path`를 고정하지 않아, 트리거를 실제로 실행하는 `supabase_auth_admin` 롤의 `search_path=auth` 설정을 그대로 물려받음 → `INSERT INTO users`가 `public.users`가 아니라 `auth.users`를 가리켜서 매 가입마다 PK 중복 에러 발생. `search_path = public` 고정 + `public.users`로 스키마 명시하여 해결 (마이그레이션 015).
- **구글 로그인 후 로그인 화면으로 튕기던 문제**: Supabase `uri_allow_list`(리다이렉트 허용목록)이 비어있어서 발생. 로컬/배포 도메인 등록으로 해결.
- **카카오 KOE205 에러**: 카카오 개발자 콘솔의 "동의항목"에서 `account_email` 스코프가 "권한 없음"으로 되어있어 발생. 필수 동의로 변경 후 해결.

### 신규 기능

## 4. Next.js 16 호환성 버그 (이번 세션에서 발견한 가장 파급력 큰 버그)

Next.js 16부터 동적 라우트의 `params`가 동기 객체가 아니라 **Promise**로 바뀌었는데, 코드베이스 곳곳이 예전 방식(`params.id` 직접 접근)으로 작성되어 있어 **프로젝트 상세/리포트/리뷰 페이지가 항상 "찾을 수 없습니다"로 표시되던 문제**를 발견하고 수정했습니다.

수정한 파일:
- `app/builder/projects/[id]/page.tsx` (서버 컴포넌트 — `async` + `await params`)
- `app/builder/requests/[id]/page.tsx` (서버 컴포넌트)
- `app/projects/[id]/report/page.tsx`, `app/projects/[id]/review/page.tsx` (구 URL 리다이렉트용)
- `app/evaluator/projects/[id]/page.tsx`, `app/evaluator/review/[id]/page.tsx`, `app/builder/reports/[id]/page.tsx` (클라이언트 컴포넌트 — `React.use(params)`로 언래핑)

API Route Handler(`app/api/**/route.ts`)들은 이미 처음부터 `Promise<params>` 방식으로 올바르게 작성되어 있어 별도 수정이 필요 없었습니다.

이 버그로 인해 "리뷰어가 지원했는데 크리에이터 화면에서 안 보인다"는 증상도 함께 발생했었는데, 실제 DB 데이터와 RLS 뷰(`project_matches_for_creator`)는 처음부터 정상 동작하고 있었음을 크리에이터 계정의 실제 액세스 토큰으로 직접 조회하여 확인했습니다. 즉 데이터 유실은 없었고, 순수하게 화면에서 못 불러오던 프론트엔드 버그였습니다.
