## 1. 랜딩페이지 — 모션그래픽 & UI

### 1-1. 리뷰어 혜택 섹션 모션그래픽 3종
- `AcrylicPortfolioVisual.tsx` (01·포트폴리오), `TrendDashboardVisual.tsx` (03·시장 트렌드), `CompensationVisual.tsx` (04·사례금 — 이후 다른 세션에서 골드 글래스모피즘 버전으로 교체됨)
- 전부 GSAP 레퍼런스 빌드를 framer-motion으로 재구현 (CDN 스크립트 로딩 없이, 이 프로젝트 전역 애니메이션 라이브러리와 통일)

### 1-2. How it works — 네온 스네이크 라인
- 얇은 하이라이트 라인 → 여러 겹의 glow 레이어를 쌓은 네온 튜브 스타일로 교체
- **버그 수정**: `vector-effect="non-scaling-stroke"`가 `stroke-dasharray`/`-dashoffset`을 디바이스 픽셀 기준으로 재해석시켜서, 대시 패턴이 중간에 한 번 더 반복되는 바람에 "선이 끊겨 보이는" 문제가 있었음 — 대시 애니메이션이 걸린 레이어에서만 해당 속성 제거해서 해결
- 코너 반경(`CORNER`)도 두꺼워진 선 폭에 맞게 재조정 (얇을 때 값 그대로 두면 굴곡부에서 stroke가 자기 자신과 겹쳐 찌그러짐)

### 1-3. 히어로 배경 이미지 프레이밍 + 전환 효과
- 기존: 이미지 하나를 정확히 반으로 잘라 크리에이터/리뷰어 각 페이지에 배치 → 인물이 화면 가장자리에 몰려 보이는 문제
- 수정: 확대(200vw→288vw) + 페이지별 오프셋을 각 인물 중심으로 재계산해서 프레이밍 개선
- **전환 효과**: 기존엔 View Transitions API로 페이지 전체를 딱딱한 판처럼 슬라이드 — 배경 이미지도 그냥 페이지의 일부로 캡처돼서 같이 밀려날 뿐이었음. 배경 이미지 요소에만 `view-transition-name`을 별도로 지정해서, 브라우저가 그 요소만 따로 떼어내 크리에이터 크롭 위치 → 리뷰어 크롭 위치로 부드럽게 팬(pan) 모핑하도록 개선. 나머지 UI(텍스트/버튼)는 기존 슬라이드 유지, 이전 페이지도 반대 방향으로 살짝 밀려나며 페이드되도록(parallax 느낌) 조정
- "시작하기" 등 CTA 버튼 관련 — 아래 4번 항목 참고 (인증 우회 버그와 얽혀 있었음)

## 2. 백엔드 파이프라인 정리

| 항목 | 내용 |
|---|---|
| **C-1** | 리뷰 제출을 `POST /api/reviews/[matchId]/submit` 하나로 통합 — 답변 insert + 매칭 상태 갱신 + 완료율 체크(`increment_completed_count` RPC가 갱신 후 실제 값을 반환하도록 변경) + 리포트 자동 생성까지 서버에서 한 번에 처리. 기존엔 브라우저가 여러 API를 순서대로 호출해서 중간에 실패하면 "완료됐는데 리포트가 안 나오는" 상태가 될 수 있었음 |
| **M-3** | 리포트 생성 성공 시 `projects.status = 'completed'`로 갱신 (C-1과 같이 처리) |
| **H-1** | `app/reviewer/account-setup`(계좌 등록 — 이미 만들어져 있었지만 어디서도 링크가 없어 도달 불가능했음)을 리뷰어 사이드바 메뉴에 연결 |
| **H-5** | 리뷰어 자진 하차(참여 취소) API(`/api/evaluator/matches/[id]/drop`) + "참여 리뷰" 목록에 취소 버튼 추가 |
| **H-2** | 마법사에서 입력받지만 저장할 컬럼이 없어 버려지던 필드(occupations/interests/targetContext/decisionFactors/validationGoal/hypothesis/targetReviewerRoles)를 `projects.extra_data JSONB`에 보존 |
| **H-4** | `/evaluator/profile` — 매칭 점수에 쓰이는 `domain_tags`(관심 분야)를 리뷰어가 직접 설정할 수 있는 화면 신규 구현 (이전엔 빈 스텁이었음) |
| **M-1** | PSF 고정 문항 매칭을 `question_text` 텍스트 비교 대신 안정적인 `question_key`로 전환 (문구가 나중에 수정돼도 매칭이 깨지지 않도록). 옛 프로젝트 호환을 위해 텍스트 매칭은 fallback으로 유지 |
| **M-5** | 마감 기한(`deadline`) 초과 시 자동 마감 처리하는 `expire-deadlines` Edge Function 신규 작성 + 매일 자정 실행 cron 등록 |
| **구 라우트 정리** | `app/projects/[id]/report\|review\|distribution`(구버전)을 신 라우트(`app/builder/reports/[id]`, `app/evaluator/review/[id]`, `/admin/distributions`)로 `redirect()` 처리. 실제로 구 경로를 참조하던 코드(`app/api/projects/[id]/join/route.ts`의 `redirectTo`)도 갱신 |

**진단/기각한 항목**: Light 타입 리포트의 verdict가 "항상 RECONSIDER로 나온다"는 보고가 있었으나, 코드 직접 검증 결과 실제로는 `null`이 나오고(Light 전용 프롬프트가 애초에 `recommendation` 필드를 요청하지 않음) UI에서도 Light 타입은 verdict를 렌더링하는 분기 자체를 안 타서 사용자에게 영향 없음 — 실제 버그 아님으로 결론, 코드 변경 없음.

**적용된 마이그레이션**: 010(`increment_completed_count` 반환값 변경) · 011(`projects.extra_data`) · 012(`review_questions.question_key`) · 013(deadline cron, 다른 세션에서 추가) · 014(`users.nickname/real_name/phone`) — 전부 원격 Supabase에 반영 완료.

## 3. 마법사(프로젝트 등록) 개선

- **관심사 키워드 중복 입력 버그** — 한글 조합(IME) 중 Enter를 누르면 조합 확정용 keydown이 한 번 더 발생해서, 마지막 글자 조각이 별도 키워드로 한 번 더 추가되던 버그. `e.nativeEvent.isComposing` 체크로 해결
- **AI 관심사 키워드 추천** 신규 추가 (`/api/interests/suggest`) — 서비스 정보 기반으로 5~8개 추천, 클릭해서 바로 추가
- **구매·사용 결정 요인** 단일 선택 → 다중 선택(`decisionFactors: DecisionFactor[]`)으로 변경
- **질문 개수 무제한화** — Light(기존 5개)/Standard(기존 5~9개) 하드캡을 전부 제거 (`Infinity`), UI 문구도 "무제한"으로 자동 전환
- **AI 추천 질문 기능 버그 수정** — Gemini 키 없을 때 쓰는 mock 응답이 "질문 추천"이라는 프롬프트 종류를 인식 못 해서 엉뚱한 객체(`key_insights` 배열)를 질문 목록인 것처럼 잘못 반환하고 있었음 — mock 응답에 질문-추천 전용 케이스 추가로 해결
- **객관식 선택지 Enter 입력 편의성** — 선택지 입력 중 Enter 치면 자동으로 다음 선택지 칸이 생기고 커서가 넘어감 (매번 "선택지 추가" 버튼 클릭 안 해도 됨)

## 4. 인증 — 소셜로그인 & 회원가입 강화

### 4-1. "시작하기" 버튼이 로그인 없이 대시보드로 바로 가던 버그
- 헤더/히어로/리뷰어 랜딩 총 5곳의 CTA가 `/builder/dashboard`, `/evaluator/dashboard`로 직접 링크돼 있었고, 두 대시보드 모두 자체 인증 가드가 없어서 비로그인 사용자도 빈 대시보드로 그냥 들어가지던 문제
- 공용 헬퍼(`lib/auth/routeToDashboard.ts`)로 통합 — 세션 없으면 로그인 화면, 있으면 role에 맞는 대시보드로 라우팅

### 4-2. 구글/카카오 소셜 로그인
- 로그인/회원가입 페이지에 버튼 추가, `/auth/callback` 라우트 신규 생성(OAuth 코드 → 세션 교환 후 role 기반 라우팅)
- Supabase Management API로 두 provider 다 활성화 + Client ID/Secret 등록 완료
- **구글 로그인 버그 수정**: Supabase의 리다이렉트 허용 목록(`uri_allow_list`)이 비어있어서 로그인 성공해도 콜백이 거부되고 로그인 화면으로 튕기던 문제 — `http://localhost:3000/**` 추가로 해결, 정상 동작 확인
- **카카오 KOE205 에러**: 카카오 개발자센터 자체 설정 문제(제가 접근 권한 없음) — "카카오 로그인 활성화" 스위치, "플랫폼 > Web" 도메인 등록(`http://localhost:3000`) 확인 필요, 사용자 쪽에서 마저 확인 중

### 4-3. 회원가입 폼 재구축
- 이메일 중복확인 / 닉네임 입력+중복확인 / 실명(텍스트, 신원 인증 아님) / 전화번호(입력만, OTP 인증은 "준비중" 비활성 버튼) / 비밀번호 확인(실시간 일치 검증)
- `users` 테이블에 `nickname`(유니크)/`real_name`/`phone` 컬럼 추가, 중복확인 전용 API(`/api/auth/check-availability`, 서비스 롤로 존재 여부만 반환)
- **전화번호 실제 SMS OTP 인증은 미구현** — NHN Cloud/Solapi 같은 유료 SMS 벤더 계약이 별도로 필요해서, 이번엔 입력 필드만 만들고 인증 로직은 보류

## 5. FindFit Agent — 실데이터 연동

- **중요 발견**: 기존 `AgentPanel`/`agentMock.ts`는 대화 흐름(4단계 phase 구조)만 기획서와 일치할 뿐, **실제 AI 호출이 전혀 없는 완전 스크립트 목업**이었음
- 경쟁사 이름 노출 정책 확인: 기획서 v3.5(이름 노출)와 기존 코드 v3.3 설계(이름 완전 비노출) 중 **v3.3(비노출) 유지로 확정** — 그래서 Phase 4(유사 서비스 레퍼런스)는 실시간 웹검색 없이 기존 방식 유지, Anthropic(Claude) 키 불필요
- **Phase 2(실시간 데이터 수집) 부분만 실제 연동**: 네이버 데이터랩 검색 트렌드 API를 실제로 호출하도록 구현(`lib/agent/naverTrends.ts`, `/api/agent/trend`) — 키 없으면 기존 정적 문구로 자동 대체. **실제 키로 테스트 완료, 진짜 데이터 확인함** (예: "헬스 웰니스 관련 검색량이 최근 6개월간 35% 감소했어요 (네이버 데이터랩)")
- **Beta 뱃지** 추가 (축소/확장 모드 헤더 둘 다)

## 6. 외부 서비스/API 연동 현황

| 서비스 | 용도 | 상태 |
|---|---|---|
| Supabase (DB/Auth) | 전체 데이터/인증 | ✅ 정상 — `.env.local` 채워넣고 실제 연결 확인 |
| 네이버 데이터랩 | Agent 실시간 검색 트렌드 | ✅ 정상 — 실제 키로 테스트 완료 |
| 구글 OAuth | 소셜 로그인 | ✅ Supabase 등록 완료, redirect 버그도 수정 — 재테스트 필요 |
| 카카오 OAuth | 소셜 로그인 | ⚠️ Supabase 쪽은 등록 완료, **카카오 개발자센터 자체 설정(KOE205) 미해결** — 사용자 확인 중 |
| Gemini | 질문/키워드 AI 추천 | ❌ **막힘** — 발급받은 키 3개 전부 무료 할당량 0. 계정/지역이 무료 티어 대상이 아닌 것으로 추정. Gemini API는 결제 계정 연결 + **최소 $10 상당 선불 크레딧 충전**이 있어야 유료 등급이 열림(GCP 일반 후불 청구와는 별개 규칙). 사용자가 결제 진행 여부 결정 대기 중 |
| Anthropic (Claude) | 고급 등급(Builder+/Launcher) 리포트 생성 | 코드는 이미 연결돼 있음(`lib/ai/claude.ts`), 키는 아직 없음 — 급하지 않음, 필요 시 나중에 |
| SMS 인증(전화번호 OTP) | 회원가입 본인확인 | 미착수 — 벤더(NHN Cloud/Solapi 등) 선정 필요 |

## 7. 알려진 이슈 / 보류 중인 결정

1. **"프로젝트 1개 제한"** — 확정 안 됨, 보류 중
2. **구 라우트(`app/projects/[id]/*`) 완전 삭제 여부** — redirect로만 처리해둔 상태, 완전 삭제는 별도 재확인 필요하다고 한 바 있음 (이번 세션에서 redirect 자체는 이미 반영됨 — 위 2장 참고)
3. **리뷰어 활동 화면(그리드형 카드 UI) 리디자인** — 숨고/크몽/체험단 사이트 참고해서 방향 논의만 하고 아직 구현 안 함. **본인이 "로직 안정된 후 진행"으로 순서 정하심** — Agent 연동(5장) 다음 순서
4. **Gemini 결제 여부 결정 대기** — 이게 막혀있으면 AI 질문/키워드 추천은 계속 mock으로만 동작 (기능 자체는 정상, 실제 AI 품질만 아직 못 봄)
5. **카카오 로그인 최종 미해결**
