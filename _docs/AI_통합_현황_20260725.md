## 1. AI 리포트 생성 — `lib/ai/generateReport.ts`

- **엔진**: Claude `sonnet` (모델 slug: `claude-sonnet-5`)
- **트리거**: 리뷰 목표 인원 달성 시 자동 생성(`app/api/reviews/[matchId]/submit`) + 크리에이터의 "재생성" 버튼(`app/builder/reports/[id]`)
- **비용 상한**: 프로젝트당 하루 3회(`report_regenerate_logs` 테이블)
- **생성 내용**: PSF/PMF 점수, 핵심 인사이트 5개, 액션 플랜 3개, 피봇/성장 시나리오 2개, 경쟁사 레퍼런스 3개, 시장 규모(TAM/SAM/SOM), 포지셔닝 맵, (베타·출시 단계일 때만) Unit Economics·GTM 전략·Scale-up 로드맵
- **현재 상태**: `ANTHROPIC_API_KEY` 크레딧 부족 → 전부 고정 mock 데이터로 대체 중. 코드 자체는 정상(2026-07-25 실키로 연결 확인, 크레딧 부족 에러까지 명확히 확인함)

## 2. 리포트 AI 챗봇 — `/api/ai-report/[projectId]/chat`

- **엔진**: Claude `haiku` (모델 slug: `claude-haiku-4-5-20251001`)
- **컨텍스트**: 저장된 `ai_reports.report_data` 전체를 매 요청마다 프롬프트에 실어 보내는 상태 없는(stateless) 방식 — 대화 이력은 서버에 저장하지 않음
- **비용 상한**: 프로젝트당 하루 15회(`report_chat_logs`), 질문 200자 제한, 응답 `max_tokens` 500
- **현재 상태**: mock (크레딧 부족)

## 3. 질문 자동추천 (2곳) — `lib/ai/index.ts`의 `generateQuestionSuggestions`

- **엔진**: Claude `haiku`
- **호출처 A**: `/api/questions/suggest` — 프로젝트 등록 마법사 **초안** 단계(아직 project_id 없음). 로그인 필요, 유저+날짜 단위 하루 10회 캡
- **호출처 B**: `/api/projects/[id]/questions/suggest` — 이미 생성된 프로젝트. project_id 단위 총 10회 캡
- **현재 상태**: mock (크레딧 부족)

## 4. 관심사 키워드 자동추천 — `/api/interests/suggest`

- **엔진**: Claude `haiku`
- **호출처**: 등록 마법사 Step3(타겟 설정) — 초안 단계라 유저+날짜 단위 하루 10회 캡
- **현재 상태**: mock (크레딧 부족)

## 5. FindFit Agent — `components/agent/AgentPanel.tsx` + `components/agent/agentMock.ts`

- **AI를 전혀 쓰지 않습니다.** 파일명 그대로 처음부터 규칙 기반 스크립트(약 286줄, if/else 분기로 단계별 정해진 문구를 노출)로 설계되어 있습니다.
- 유일하게 실데이터를 쓰는 부분은 **트렌드 한 줄 코멘트**(예: "OO 관련 검색이 최근 6개월간 X% 증가했어요")이며, 이건 Claude/Gemini가 아니라 **네이버 데이터랩 API**(`lib/agent/naverTrends.ts`)를 호출합니다.
  - `NAVER_CLIENT_ID`/`NAVER_CLIENT_SECRET` 설정되어 있고, 2026-07-25 기준 실제 API 호출로 정상 데이터 수신 확인함(예: "스타트업 서비스" 검색 비율 6개월 추이 실데이터).
  - 두 키가 없으면 카테고리별 고정 문구(`FALLBACK_TEXTS`)로 자동 대체.
- **결론**: Anthropic 크레딧과 무관하게, Agent의 대화 흐름 자체는 원래 설계부터 AI가 아니라서 크레딧을 충전해도 이 부분은 바뀌지 않습니다. 실제 AI 대화형으로 바꾸려면 별도 개발이 필요합니다(아직 미착수).

## 6. Gemini — `lib/ai/gemini.ts`

- 이전에는 리포트/챗봇/자동추천 전부 Gemini를 썼으나, 2026-07-25 작업으로 **전부 Claude로 전환 완료**.
- `gemini.ts` 파일 자체는 삭제하지 않고 남겨뒀지만, 실제 호출 경로(`callGemini`)는 이 파일 자기 자신 외에 아무도 참조하지 않음(grep으로 확인) — 즉 죽은 코드.
- `.env.local`의 `GEMINI_API_KEY`는 더 이상 어떤 기능에도 영향 없음.

## 7. 알려진 이슈 (이번에 발견하고 수정한 것)

- Claude Messages API는 Gemini와 달리 강제 JSON 모드가 없어서, "JSON만 반환하라"고 프롬프트에 지시해도 가끔 &#96;&#96;&#96;json ... &#96;&#96;&#96; 코드펜스로 감싸서 응답함 → 그대로 `JSON.parse`하면 "Unexpected end of JSON input"이라는 애매한 에러만 남음. `lib/ai/claude.ts`에 코드펜스 제거 로직(`extractJson`)을 추가하고, API 에러 응답도 명확히 던지도록 수정함(이 덕분에 "크레딧 부족"이라는 진짜 원인을 바로 확인할 수 있었음).
